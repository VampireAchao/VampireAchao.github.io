is-a.dev

is-a.dev is a service that allows developers to get a sweet-looking .is-a.dev subdomain for their personal websites.

让你的个人站点，穿上一件“甜甜的”马甲：is-a.dev 的自我介绍

如果域名会说话，那 is-a.dev 一定是那种一开口就让人觉得“哇，你好会取名”的角色。

它不想当一个高高在上的顶级域名，也不想当一个冷冰冰的 DNS 工具人——它更像一位热情的领路人，站在开发者的路口挥手：

“嘿，开发者！来拿一个 sweet-looking 的 .is-a.dev 子域名吧，给你的个人网站一个更体面的门牌号。”

而这句自我介绍，正是它在仓库的 description 里写下的承诺：

Grab your own sweet-looking ‘.is-a.dev’ subdomain.

它说到做到，而且把整个流程摊开在 GitHub 上，让你看得见它的认真、也摸得着它的透明。

这位“域名分发官”到底在做什么？

is-a.dev 的工作很简单，也很可爱：

  • 你有个人网站
  • 你希望它拥有一个更有辨识度、开发者味儿更浓的地址
  • 它就把 .is-a.dev 这个尾巴递给你，让你拥有一个专属的子域名

它不替你写博客、不替你做作品集，但它会把你的网站“介绍”得更像你：开发者、创作者、折腾爱好者。

它的性格也写在 README 里，直白得像一封公开信：

is-a.dev is a service that allows developers to get a sweet-looking .is-a.dev subdomain for their personal websites.

它很爱热闹：Announcements 都在 Discord

is-a.dev 不是那种“出了事才发朋友圈”的项目……好吧，它在 README 里承认了一半：

  • 并不是所有公告都会发在 GitHub
  • 但是它保证：Discord 一定会发
  • 至于 GitHub——只有“严重事故”才会置顶通告

所以它认真邀请你去它的 Discord 频道串门：

Please join our Discord server for announcements, service updates, and downtime notifications regarding the service.

它就像在门口贴了一张纸条：

“重要消息别错过，来 Discord 听我唠嗑。”

Register：想要 .is-a.dev？来走流程吧

is-a.dev 这位“分发官”很讲规矩，申请方式清清楚楚写着：

If you want a visual guide, check out this blog post.

它甚至还贴心地给你准备了一个“可视化指南”的博客链接，让你不用光看文字想象流程。

接下来，就是正餐——注册流程：

  • Fork 这个仓库
  • 按照它的 documentation 做（它把入口摆在了那儿：https://docs.is-a.dev）
  • 然后打开一个 PR
  • 它会来 review
    • 而且它会提醒你一句很“严肃”的话：
      Keep an eye on it in case changes are needed!
    • 如果它提出修改意见，你不改——它也不惯着：
      you will be rejected
  • 一旦 PR 合并，它会迅速行动：
    你的 DNS records 几分钟内就会发布
  • 最后，它会像朋友一样拍拍你肩膀：
    Enjoy your new .is-a.dev subdomain!
    还顺手补一句：喜欢的话给个 star ⭐️ 吧

把流程写成“命令”：一套动作行云流水

README 里写得是步骤，但我们可以把它理解成一种“开发者式仪式感”：

1）Fork 仓库（你和 is-a.dev 的第一次握手）

2）阅读并按文档操作（你开始了解它的脾气）

3）发起 PR（你把申请递给它）

4）盯住 PR（它可能会“挑刺”，但那是它认真）

5）PR 合并后等几分钟（它去给 DNS 发指令）

6）你拥有新门牌号（它完成使命，也替你开心）

它也有底线：Report Abuse

is-a.dev 不是“谁来都给”的慈善家，它更像一个有秩序的社区管理员。

如果你发现有人滥用子域名、或者做了违反 TOS 的事情，它会请你来“敲门报告”：

If you find any subdomains being abused or breaking our TOS, please report them by creating an issue with relevant evidence.

它把入口也直接给你准备好：
创建一个带证据的 issue，贴上 abuse 的标签，它就会来处理。

背后有人罩着：Cloudflare Project Alexandria

is-a.dev 也不是一个孤零零的��项目，它在 README 里郑重鸣谢它的支持者：

We are supported by Cloudflare’s Project Alexandria sponsorship program, we would not be able to operate without their help!

它的语气不像官话，更像是那种认真说“没有你我真不行”的感谢。

然后它把 Cloudflare 的 Logo 端端正正放出来，像把赞助人的名牌擦得锃亮。

结尾：is-a.dev 的小心愿

它给你的不只是一个子域名，更像是一种身份标签：

  • 你是开发者
  • 你有作品、博客、页面、想法
  • 你想把它们摆在一个更“像开发者”的门牌下

于是它把 .is-a.dev 递给你——甜甜的、清爽的、非常合适的。

如果你已经在准备 Fork、准备 PR、准备把自己的网站搬到新地址上——那就去吧。

它会在那边等你。
