NewPipe：Android 上“轻、自由、私密”的流媒体前端

如果你希望在 Android 上自由、轻量地浏览和收听主流视频与音频内容，同时尽量减少对专有生态和跟踪服务的依赖，NewPipe 是一个值得认真体验的项目。它是一个开源、轻量的“流媒体前端”，以“尽可能私密与可控”为原则，支持多种服务源，功能齐备，且不会强制你登录某个账号就能订阅或管理内容。

  • 项目主页：newpipe.net
  • 仓库地址：TeamNewPipe/NewPipe
  • 许可证：GPLv3
  • 简述：A libre lightweight streaming front-end for Android.

NewPipe 的代码正在进行“重构大改版”，官方推荐对新功能的贡献基于 refactor 分支进行；当前主线代码进入维护模式并以修复为主。与此同时，NewPipe 仍作为“开源自由替代”的代表，持续提供可靠而丰富的使用体验。

为什么用 NewPipe

  • 自由开源与轻量设计
    使用自由软件许可证，拒绝专有服务绑定。应用和抽取层都不依赖诸如 Google Play Services 的专有库，使其在更多设备和 ROM 场景下可用。

  • 多服务支持与可扩展
    除了较成熟的 YouTube 支持，NewPipe 同时覆盖了多种视频与音频服务源，包括 YouTube Music、PeerTube、Bandcamp、SoundCloud、media.ccc.de 等。

  • 私密和可控
    目标是提供尽可能私密、匿名的使用体验；不收集数据（除非你明确同意）。你可以本地订阅与分组管理，无需登录某个专有账号。

  • 强功能与友好体验
    从 4K 播放、后台音频、弹窗画中画、直播与字幕、历史与播放列表，到订阅分组、通知与下载，常用能力一应俱全。

支持的服务（节选）

  • YouTube 与 YouTube Music
  • PeerTube 及其实例
  • Bandcamp
  • SoundCloud
  • media.ccc.de

YouTube 因生态体量与用户覆盖，更受关注与维护。如果你熟悉其他平台，欢迎帮助完善与贡献，让生态更加丰富。

核心功能亮点

  • 最高至 4K 播放
  • 后台音频（仅拉取音频流以节省数据）
  • 弹窗模式（画中画）
  • 直播观看
  • 字幕/CC 显示与隐藏
  • 搜索（YouTube 支持指定内容语言）
  • 排队与本地播放列表
  • 视频信息面板与相关视频、评论显示/隐藏
  • 频道浏览与订阅（无需登录）
  • 订阅频道的新内容通知与频道分组
  • 分组生成的视频 feed 浏览
  • 观看历史
  • 收藏与播放列表搜索（远程列表与本地列表皆可）
  • 下载视频/音频/字幕
  • 支持跳转到 Kodi
  • 年龄限制内容观看/屏蔽

使用与安装

NewPipe 提供多种安装/更新方式。官方建议普通用户优先使用 F-Droid 仓库方式。

  • 推荐：添加 NewPipe 的自定义 F-Droid 仓库并安装
    教程：https://newpipe.net/FAQ/tutorials/install-add-fdroid-repo/

  • 直接下载 APK（GitHub Releases）并核验签名指纹
    https://github.com/TeamNewPipe/NewPipe/releases

  • 通过 F-Droid 更新（速度较慢，因为 F-Droid 需自行识别构建并推送）

  • 自行构建调试版 APK（适合开发者）

  • 下载指定 PR 的 APK（若你只对某个 PR 中的特定修复/功能感兴趣）

切换安装来源的建议流程：

  1. 设置 > 备份与恢复 > 导出数据库（保留历史、订阅、播放列表）
  2. 卸载旧版本
  3. 从新来源安装最新 APK
  4. 设置 > 备份与恢复 > 导入数据库（恢复数据）

使用体验与小贴士

  • 隐私与匿名体验
    项目的隐私政策明确：应用不会在你未同意的情况下收集任何数据。你可在设置中导出/导入数据库，实现跨来源安装后的完整迁移。

  • 订阅与分组
    无需登录即可订阅频道，并根据自身偏好建立分组、生成 feed。对于日常追更，这是轻量但实用的方案。

  • 播放与下载
    NewPipe 同时兼容在线播放与本地下载使用场景。对于数据流量敏感或离线收听需求，这是利好。

  • 弹窗画中画与后台音频
    当你想一边浏览其他内容，一边继续播放音频或视频，弹窗与后台模式让体验更顺畅。

开发与贡献

NewPipe 目前正在进行较大规模的重构与重写。如果你想为新功能贡献代码，官方推荐在 refactor 分支上进行；当前主代码库以维护与修复为主。

贡献不限于代码：翻译、设计、文档与清理都欢迎。社区提供了 IRC 与 Matrix 的交流渠道，也在 Weblate 上进行翻译协作。

交流与协作渠道：

  • IRC Chat：#newpipe
  • Matrix：#newpipe:matrix.newpipe-ev.de
  • 翻译协作（Weblate）：Hosted Weblate 平台
  • 文档站：teamnewpipe.github.io/documentation

重要声明与合规

  • Beta 状态
    应用目前处于 Beta，可能遇到 bug。若发现问题，请按 issue 模板在 GitHub 仓库提交 issue。

  • Google Play 上架
    将 NewPipe（或任何 fork）发布到 Google Play 商店会违反其条款与条件；请勿将其上架到 Play。

  • 法律与服务 API
    NewPipe 的工作机制是尽可能用各服务的公开 API 获取数据；在官方 API 受限（如 YouTube）或专有时，抽取层会采用非官方方式以满足使用需求，同时尽可能遵循开源与隐私原则。

许可证与项目愿景

NewPipe 使用 GPLv3 许可。它是自由软件：你可以自由使用、研究、分享和改进。项目的宗旨是为用户提供一个私密、匿名、自由的流媒体前端选择。

结语

NewPipe 不是“另一个客户端”，而是把“自由与隐私”融进日常流媒体使用中的一次尝试。它让订阅、观看、收听与下载这件事，从专有生态的捆绑中抽离出来，以轻量、可控的方式回到你自己的设备与习惯。
如果你追求“少打扰、少绑定、多可控”，那就从 F-Droid 安装 NewPipe，导入你的数据，建立订阅组，打开后台音频与弹窗模式，给自己的流媒体体验一次“轻巧且自由”的升级。
