Palmier Pro：当视频编辑器开始学会与 AI 并肩创作

如果说传统视频编辑器像一位经验老到、手法稳健的剪辑师，那么 Palmier Pro 更像是一位坐在你身边、会思考、会协作、还懂现代创作节奏的新搭档。

它不是在旁边冷冰冰地等待你下达命令，也不是把 AI 生硬地贴在软件表面当作一个噱头功能。Palmier Pro 的气质更像是这样一种存在：你把素材丢进时间线，它抬起头看你一眼，轻声说，来吧，我们一起把这个故事剪出来。

Palmier Pro 是一个面向 macOS 的开源视频编辑器，它的核心方向非常明确：为 AI 而生。项目的描述写得很直接，macOS video editor built for AI。README 里则进一步把它的身份说得更完整：这是一个开源的 Mac 视频编辑器，你和你的智能体可以在同一条时间线里一起生成、一起编辑视频。

这句话很重要。

因为 Palmier Pro 并不是单纯让 AI 帮你“生成一段东西”，而是试图让 AI 真正进入视频创作的工作流，进入时间线，进入项目上下文，进入人与工具之间那种反复推敲、不断修改、逐步成形的过程。它不是站在门外递素材，而是走进剪辑室，坐到你旁边。

一个从头用 Swift 写出来的视频编辑器

Palmier Pro 不是在旧世界上加补丁，而是从头开始构建。

项目在 README 中明确提到，它是用 Swift 从零打造的。它把 Premiere Pro 视作北极星，同时又想走出自己的路：把 AI 真正整合进视频编辑流程，而不是把它当成一个外挂式插件。这种选择很有意思，也很有野心。

你可以把它理解成一种态度。

它没有对着传统编辑器毕恭毕敬地说，我来补充一点智能功能吧；它更像在认真地回答另一个问题：如果今天重新设计一款视频编辑器，并且默认 AI 就在现场，那它应该长成什么样子？

Palmier Pro 给出的答案，是时间线依然重要，专业剪辑的骨架依然重要，但创作关系变了。过去是人独自操作工具，现在是人和 agent 一起在项目里工作。过去时间线是单人舞台，现在时间线更像一个协作现场。

它最迷人的地方，在于 AI 不是装饰，而是住进了时间线

Palmier Pro 在 README 里用了一个非常核心的表达：You and your agent can generate and edit videos together inside the timeline.

这不是一句普通的宣传文案，而是整个产品的灵魂。

很多软件会告诉你，它支持 AI。可 Palmier Pro 更像是在说，我不是“支持”AI，我是为它留了位置、留了座位、留了入口。我不是把 AI 放在边栏里让它偶尔出来表演一下，我是让它和你一起待在时间线里。

这就像一间剪辑室里，原本只有你一个人盯着素材反复拖拽、切分、拼接。Palmier Pro 把门推开，请进来一个新的搭档。这个搭档能理解你的项目，能参与编辑，能协助生成内容，也能和你对着同一条时间线商量下一步要怎么走。

于是视频创作不再只是单线程的劳动，而变成了协同写作、协同剪辑、协同试错。

内建生成式 AI，把想象力直接送进剪辑流程里

Palmier Pro 在 README 中提到，它内建了生成式 AI，并且可以在时间线编辑器内部调用当前最先进的一些模型来生成视频和图片，例如 Seedance、Kling、Nano Banana Pro。

这意味着什么？

意味着灵感不再需要频繁跨软件跳转。过去你可能需要在一个生成工具里产出素材，再导出、整理、导入剪辑器，然后继续编辑。这个过程像把创作切成几块，灵感在软件之间来回搬家，搬着搬着，热度就散了。

Palmier Pro 试图做的，是让“生成”和“编辑”坐在同一张桌子上。

你在时间线上工作时，不必把生成式 AI 当作远方的工厂，而可以把它当作现场的助手。你需要一个镜头，它能帮你补；你需要一张图，它能迅速生；你想把已有内容继续推进，它就直接在编辑上下文里参与。创作不再是先生成、再剪辑的接力赛，而更像是一场持续发生的对话。

这让整个流程拥有了一种更现代的节奏感。不是中断，不是切换，不是重启，而是流动。

它还愿意把门打开，让你的智能体直接接入

Palmier Pro 还有一个很有辨识度的特征：它不只内建 agent，也愿意和你外部使用的 agent 协作。

README 里写得很清楚，它可以通过 MCP 连接 Claude、Codex、Cursor，也可以直接使用应用内的 agent，与用户一起处理同一个项目。这一点尤其有想象力。因为它不是在争夺你的工作台，而是在主动接入你的工作方式。

Palmier Pro 仿佛在说：

你已经有自己的 AI 搭档了？很好，把它也带进来。
你习惯在 Claude 里思考，在 Codex 里操作，在 Cursor 里组织上下文？没问题，我们一起工作。

这种开放感非常迷人。它没有把自己包装成一个封闭王国，而是把自己建设成一个协作空间。你打开软件，它不仅是编辑器，也是一个能与 agent 通信的现场。

这种产品思路，非常贴近 AI 时代的软件形态。

它并不把一切都锁起来，反而在开放和商业之间做了清晰划分

Palmier Pro 的 FAQ 很坦率。

它说得很清楚：视频编辑器本体在不包含生成式 AI 功能的前提下，是完全开源的。MCP server 和 agent chat 也是开源的。唯一闭源的部分，是生成式 AI 的处理流程。

这种表达其实很有分寸。

它既没有模糊边界，也没有打“全都开源”却暗藏保留项的擦边球，而是很诚实地告诉你：哪些部分开放，哪些部分出于产品和服务的考虑暂不开放。这种清楚，反而能建立信任。

同样在费用方面，它也给出了很直接的答案：编辑器本身是免费的，不需要登录就可以下载使用，可以把它当作视频编辑器来工作，就像使用 CapCut 或 Adobe Premiere 一样。MCP server 也可以免费使用，方便大家开始实验和接入自己的 agent。只有生成式 AI 功能需要登录和订阅。

这是一种很现实也很友好的设计。

它先让你真正碰到产品、用上产品、理解它的工作方式，再决定你要不要为更深入的 AI 生成能力付费。对创作者来说，这比一上来就筑起高门槛，更容易建立关系。

它很专注，甚至有点倔强

Palmier Pro 目前支持的平台非常明确：只支持 macOS 26 Tahoe，并且只支持 Apple Silicon。

你可以把这理解成限制，也可以把它理解成一种聚焦。

Palmier Pro 没有急着讨好所有平台、照顾所有硬件，而是把力量集中在一个最适合自己表达的舞台上。它像一位挑剔的导演，只选择最能配合自己节奏的摄影棚。这样的选择，通常意味着产品想把体验做深，而不是把覆盖面铺平。

尤其考虑到它是 Swift 原生构建的 Mac 视频编辑器，这种平台专注其实非常合理。它让整个项目在技术路径和产品体验上，都有更清晰的中心。

它甚至在视觉搜索上，也悄悄准备了一点聪明劲

除了主 README，仓库里的 models/README.md 也透露了一个很有意思的细节：Palmier Pro 使用 CLIP 类模型来支持视频编辑器中的语义搜索能力，让用户和 agent 可以对素材进行视觉搜索。

这件事听起来很低调，但其实很有价值。

因为视频素材最常见的痛点之一，就是“有很多，但找不到”。镜头堆在那里，像一间塞满胶片的仓库，你知道东西在里面，却不知道它究竟藏在哪一排架子后面。Palmier Pro 试图用本地运行的模型，把这间仓库变成一个能理解你描述的档案管理员。

它使用的是 Google 的 SigLIP 2，通过 Apple 的 Core ML 框架在本地运行。模型不会直接打包进应用，而是在运行时下载相关制品，并进行校验。这透露出产品另一层野心：它不只是想让 AI 帮你生成内容，也想让 AI 帮你理解和组织已有内容。

于是 Palmier Pro 不只是会“做”，也开始会“找”、会“看”、会“理解”。

这就让它更像一个真正参与创作过程的搭档，而不只是一个素材制造机。

如果把它当作一位创作伙伴，它的性格大概是这样的

Palmier Pro 像一个年轻但目标极其清晰的人。

它身上有专业编辑器的骨架，说明它知道视频创作不是儿戏；
它愿意从 Swift 原生重建，说明它不满足于修修补补；
它把 AI 放进时间线，说明它不想只做一个旁观者；
它开放 MCP 接口，说明它不怕协作，反而欢迎协作；
它清楚划分开源与闭源边界，说明它没有故作暧昧；
它只专注 macOS 和 Apple Silicon，说明它宁愿先把一个方向做锋利。

它不像那种四处迎合、什么都想做一点的软件，更像一个已经决定自己要成为什么的人。

这种项目，往往最值得关注。

因为它不是在追赶潮流，而是在尝试定义一种新的创作关系：人、时间线、素材、模型、agent，不再是彼此割裂的几个系统，而是逐步汇成同一个工作现场。

为什么 Palmier Pro 值得被认真看待

这几年，AI 工具层出不穷，很多产品都在试图挤进创作者的工作流。但真正困难的从来不是“接入一个模型”，而是“让模型自然地参与工作”。

Palmier Pro 的可贵之处，在于它没有停留在表面功能层。它不是在说，我这里也能生成一下；它是在努力回答，AI 与视频编辑究竟应该如何共同存在。

它的答案是：

不是外挂式存在，
不是临时性帮忙，
不是切出去再切回来，
而是一起待在时间线里，一起做项目。

这会给创作者带来很不一样的感受。

当工具不再只是被动执行，而是能参与上下文、连接 agent、融入创作节奏时，视频编辑这件事本身就开始变化了。它变得更像编排、指挥、协作、对话，而不是单纯机械操作。

Palmier Pro 正在朝这个方向认真前进。

快速认识 Palmier Pro 的使用入口

如果你想快速理解 Palmier Pro 最关键的协作能力，可以先记住这几件事：

  1. 它是一款面向 macOS 的开源视频编辑器
  2. 它的定位是为 AI 而生
  3. 它允许你和 agent 一起在时间线里生成与编辑视频
  4. 它支持通过 MCP 对接 Claude、Codex、Cursor
  5. 它内建生成式 AI，支持在时间线中生成视频和图片
  6. 编辑器本体免费可用，生成式 AI 功能需要登录和订阅
  7. 当前平台要求是 macOS 26 Tahoe 与 Apple Silicon

结语

Palmier Pro 最吸引人的地方，不只是它是一款开源视频编辑器，也不只是它内建了生成式 AI，而是它试图让视频创作这件事进入一种新的状态。

在这个状态里，时间线不再只是素材的排列轨道，也成了人和 agent 共同创作的会面地点。
在这个状态里，编辑器不再只是命令执行器，也成了一个可连接、可协同、可扩展的创作节点。
在这个状态里，AI 不再像舞台边缘的嘉宾，而是被请到了台中央，真正参与镜头、节奏、结构与表达。

Palmier Pro 就像一个正在成长中的新型剪辑伙伴。它坐在 macOS 上，穿着 Swift 做成的外衣，手里拿着时间线，另一只手则伸向 AI 世界。它不喧哗，却很明确；不保守，却也不混乱；不想当一件花哨的演示品，而是想成为一个真正能工作的工具。

如果未来的视频编辑注定要与 AI 深度共创，那么 Palmier Pro 看起来已经提前坐在了那个未来的门口，回头朝我们招了招手。

它像是在说：

故事你来讲。
节奏我们一起找。
镜头已经在路上了。
